# Fitness Tracker Server

## Overview
The Fitness Tracker Server is the backend API for the Fitness Tracker application. It handles user authentication, trainer management, booking sessions, and payment processing. The server is built using Node.js, Express, and MongoDB.

## Key Features
- **RESTful API**: Provides endpoints for user authentication, trainer management, and bookings.
- **User Authentication**: Secure registration and login using JWT (JSON Web Tokens).
- **Payment Handling**: Integrates with Stripe to process payments securely.
- **Data Management**: Uses MongoDB for storing user and booking data.
- **Error Handling**: Comprehensive error handling for API requests.

## Technologies Used
- **Node.js**: JavaScript runtime for building the server.
- **Express**: Web framework for building RESTful APIs.
- **MongoDB**: NoSQL database for data storage.
- **Mongoose**: ODM for MongoDB (not used in this project as per requirements).
- **Stripe**: For payment processing.
- **Firebase**: For authentication.
- **JWT**: For secure authentication.

## Usage
- Ensure the server is running and accessible.
- Use tools like Postman or Insomnia to test the API endpoints.
